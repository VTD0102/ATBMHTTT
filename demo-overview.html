Kịch Bản Tấn Công & Phòng Thủ

Demo tái hiện một cuộc tấn công Social Engineering + Ransomware nhắm vào doanh nghiệp TMĐT nhỏ — từ bước lừa đảo đầu tiên đến khi phục hồi hoàn toàn dữ liệu.

1
Social Engineering — Lừa Đảo
MITRE T1566 Phishing · T1204 User Execution

Kẻ tấn công dựng trang marketing landing-web giả mạo phần mềm quản lý dự án "ProManager Suite". Giao diện chuyên nghiệp có đầy đủ testimonials, bảng giá, feature list — đủ để đánh lừa người dùng thiếu kinh nghiệm bảo mật.

  • Nạn nhân lướt web, thấy quảng cáo phần mềm hấp dẫn
  • Truy cập landing page (localhost:5173) — trông hoàn toàn hợp lệ
  • Bấm "Download Free Trial" → tải ProManagerSuite.exe về máy
  • File thực chất là ELF binary ~14MB, đặt tên .exe để gây nhầm lẫn hệ điều hành
2
Initial Access — Xâm Nhập
MITRE T1204.002 Malicious File

Nạn nhân tự chạy file exe — không cần leo thang đặc quyền. Ứng dụng hiển thị giao diện trắng/indigo hoàn toàn hợp lệ, qua 3 màn hình trước khi payload kích hoạt. Nạn nhân không nhận ra bất thường.

  • Splash Screen — logo, loading animation
  • Activation Screen — nhập API key (bất kỳ giá trị nào đều được chấp nhận)
  • Verifying Screen — fullscreen, animation "Đang xác thực..." giả 3 giây
  • Dashboard — sidebar navigation, 6 project cards, biểu đồ thống kê
  • Thread ngầm bắt đầu đếm 10 giây kể từ khi Dashboard load xong
3
Execution — Thực Thi Payload Ransomware
MITRE T1486 Data Encrypted for Impact

Sau 10 giây kể từ khi Dashboard xuất hiện, payload kích hoạt ngầm trên background thread. Toàn bộ quá trình mã hóa và gửi key về C2 hoàn tất trước khi nạn nhân kịp phản ứng.

  • Tạo Fernet key 256-bit ngẫu nhiên (AES-128-CBC + HMAC-SHA256)
  • Quét toàn bộ file trong sandbox, bỏ qua .exe .py .sh .so .dll
  • Mã hóa từng file thành *.encrypted, đổi tên gốc thành *.demo_original
  • Lưu .ransom_key tạm, sau đó gửi về C2 server (POST localhost:8888/register)
  • Nếu C2 nhận thành công → xóa .ransom_key khỏi máy nạn nhân
  • fake_ransom.py chiếm toàn màn hình — đòi 59,000,000 VND, đồng hồ đếm ngược
4
Detection — Phát Hiện
Defender: Scanner + Behavior Monitor + YARA

Defender GUI chạy nền liên tục giám sát thư mục shop_data/. Nhiều tầng phát hiện độc lập cùng lúc báo động — không tầng nào phụ thuộc tầng khác, đảm bảo zero false negative trong demo.

  • behavior_monitor (inotify): phát hiện file .encrypted xuất hiện ngay lập tức, kill process nếu file gốc bị xóa hàng loạt
  • scanner: đo Shannon entropy — file .encrypted đạt ~8.0 bits (gần tối đa lý thuyết)
  • static_analyzer: quét string nguy hiểm + 3 YARA rules nhận diện Fernet token header (gAAAAA)
  • Flag RANSOMWARE_DETECTED.flag tự động tạo, event feed trong GUI cập nhật real-time
5
Incident Response — Ứng Phó & Khôi Phục
NIST SP 800-61: Containment → Eradication → Recovery

Đội phòng thủ thực hành đủ 3 giai đoạn ứng phó sự cố với công cụ thực tế. Kết quả hoàn toàn phụ thuộc vào việc backup sạch có tồn tại trước khi tấn công hay không.

  • Containment: tạo NETWORK_ISOLATED.flag, quarantine file exe vào thư mục quarantine/
  • Eradication: static_analyzer phân tích exe, xem behavior log, kiểm tra C2 log
  • Recovery — Cách 1: restore từ backup sạch tạo trước tấn công (luôn dùng được)
  • Recovery — Cách 2: nếu file .ransom_key còn trong sandbox → chạy decryptor.py
  • Recovery — Cách 3: lấy key từ c2_victims.json rồi decrypt thủ công
  • Unlock code ransom note cho demo: DEMO-SAFE-2024-TMDT

scanner.py
Entropy Scanner — Phát Hiện File Mã Hóa

Phát hiện file bị mã hóa dựa trên Shannon entropy — dữ liệu mã hóa/nén có entropy cao hơn văn bản thông thường. Không phụ thuộc signature cố định, hoạt động với ransomware chưa biết.

  • Tính Shannon entropy: H = -Σ p(x)·log₂p(x) cho từng file
  • Ngưỡng trên 7.5 bits → CRITICAL, từ 6.5 đến 7.5 → HIGH/MEDIUM
  • Phát hiện extension đáng ngờ: .encrypted, .locked, .crypto, .enc
  • Quét signature đặc trưng của simulator trong file .py và .sh
  • Output màu theo mức độ, hỗ trợ recursive scan thư mục
python3 defender/scanner.py shop_data/

static_analyzer.py
Phân Tích Tĩnh — String Scan + YARA

Phân tích file trước khi chạy để phát hiện đặc điểm malware mà không cần thực thi. Kết hợp string matching thủ công và 3 YARA rules chuyên biệt.

  • Quét string nguy hiểm: hardcoded key, lệnh shell, tiến trình hệ thống
  • Phát hiện import đáng ngờ: subprocess, os.system, cryptography, socket
  • YARA Rule 1: nhận diện source code simulator qua chuỗi đặc trưng
  • YARA Rule 2: phát hiện Fernet token header (byte magic gAAAAA)
  • YARA Rule 3: phát hiện HTML ransom note qua nội dung đặc trưng
  • Tự động fallback nếu không cài yara-python — chỉ dùng string scan
python3 defender/static_analyzer.py manager-agent/fake_manager.py

behavior_monitor.py
Giám Sát Hành Vi — inotify Real-time

Giám sát filesystem real-time bằng Linux inotify. Phản ứng trong mili-giây khi có thao tác file bất thường — không cần đợi scan định kỳ. Lớp phòng thủ nhanh nhất trong hệ thống.

  • Subscribe inotify events: IN_CREATE, IN_DELETE, IN_MODIFY, IN_MOVED_FROM
  • Chụp snapshot danh sách file trước khi chạy installer để so sánh
  • Phát cảnh báo ngay khi file .encrypted xuất hiện trong thư mục giám sát
  • Kill process target nếu phát hiện file gốc bị xóa đột ngột hàng loạt
  • Ghi log chi tiết từng event có timestamp chính xác đến microsecond

Quản Lý Backup — tar.gz + Timestamp

Tuyến phòng thủ hiệu quả nhất trước ransomware — backup sạch trước tấn công cho phép khôi phục hoàn toàn dù key đã bị gửi về C2. Minh họa tầm quan trọng của quy tắc 3-2-1 backup.

  • Tạo snapshot shop_data/ dạng tar.gz có timestamp: backup_YYYYMMDD_HHMMSS.tar.gz
  • Bỏ qua hoàn toàn *.encrypted, *.demo_original, .ransom_key khi backup

  • List backup theo thứ tự thời gian kèm kích thước file
  • Restore 1 lệnh — tự xóa file .encrypted sau khi restore thành công
